## 2. 現在の天気サマリー

| 要素 | 内容 | 備考 |
|---|---|---|
| 現在気温 | `currentTemp`（℃） | WeatherData |
| 体感温度 | `apparentTemp`（℃） | ユーザーが最も感じる温度 |
| 天気アイコン | 晴れ・曇り・雨などのビジュアル | `weatherCode` から決定 |
| 湿度 / 風速 | 補足情報として小さく表示 | humidity / windSpeed |

---

## 3. パーソナライズ服装提案エリア ⭐ 核心機能

### 3-1. Gemini アドバイス文
- ユーザーの体質設定（暑がり・標準・寒がり）＋気象データをプロンプトとしてGemini APIへ送信
- 返ってきた自然文アドバイスを表示
  - 例（暑がり）: 「今日は25℃と蒸し暑め。Tシャツ＋軽い短パンで十分です。」
  - 例（寒がり）: 「気温は18℃ですが風が強め。薄手のダウンを羽織ることをおすすめします。」

### 3-2. 服装イラスト / アイコン
- Geminiの提案をもとに、上半身・下半身・アウターなどをアイコンで視覚化
- **テキストを読まなくても服装が把握できる** ことを最優先（US-03）
- 暑がり設定 → 薄着構成のアイコン
- 寒がり設定 → 厚着・重ね着構成のアイコン

---

## 4. 時間軸リスト（出発 / 昼 / 帰宅）

> US-04「朝・昼・夜の気温変化に対応した服装を知りたい」

各カラムに以下を横並びで表示:

| 時間帯 | 表示内容 |
|---|---|
| 🌅 出発 | 予想気温・天気アイコン・服装アイコン |
| ☀️ 昼 | 予想気温・天気アイコン・服装アイコン |
| 🌆 帰宅 | 予想気温・天気アイコン・服装アイコン |

- `hourlyForecast`（24時間分の体感温度リスト）から各時間帯の値を抽出
- 1日の寒暖差が **10℃超** の場合 → 「脱ぎ着しやすい重ね着」構成を優先提示（spec §5）

---

## 5. 帰宅冷え込みアラート

> US-05「帰宅時に冷え込みそうな日は事前に教えてほしい」

- 帰宅時気温が出発時より大幅に低い場合、アドバイスエリアに警告バナーを表示
- **寒がり設定のユーザー** は閾値を低めに設定（例: 5℃差でも表示）
- 表示例: 「⚠️ 今夜は帰宅時に急冷えが予想されます。一枚多めに持ち出して！」
